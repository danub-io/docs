Bem-vindo à documentação do **CTECH (TechReveal)**, um ecossistema automatizado para análise, comparação e curadoria de hardware com IA.

---

## Visão Geral

O CTECH é dividido em dois repositórios que compartilham o mesmo banco **Turso (SQLite)**:

| Repositório | Stack | Papel |
|-------------|-------|-------|
| **ctech_be** | Next.js 16, Turso, Pino, AI SDK, Tailwind v4 | Backend/Painel: Automação, Scrapers, Processamento IA |
| **ctech_fe** | Astro 6, React 19, Tailwind v4, Turso, Zod v4 | Frontend/Público: Interface de alta performance otimizada para SEO |

```
ctech_be (Server Actions) → Turso DB (SQLite) ← ctech_fe (Astro SSR)
```

## Arquitetura — Módulos de Automação

### Backend (ctech_be)

O backend é dividido em 9 módulos independentes (M1-M9):

| Módulo | Nome | Função |
|--------|------|--------|
| M1 | Entrada | Ingestão e detecção de duplicidade semântica via IA |
| M2 | Descoberta | Busca de links de reviews técnicas |
| M3 | Extração | Processamento de reviews e extração de specs via IA |
| M4 | Consolidação | Unificação de reviews com nota bayesiana |
| M5 | Preços | Monitoramento de preços em lojas |
| M6 | Conferência | Auditoria de links afiliados e estoque |
| M7 | CMS | Gerenciador central do catálogo de produtos |
| M8 | Configurações | Painel de configurações globais (IA, Scrapers, Logs) |
| M9 | Documentação | Visualizador de documentação Markdown integrado |

### Frontend (ctech_fe)

| Rota | Descrição |
|------|-----------|
| `/` | Home com produto em destaque e tendências |
| `/laptops` | Lista de laptops com filtros |
| `/compare` | Comparação lado a lado de produtos |
| `/community` | Feed com reviews da comunidade |
| `/reviews/[slug]` | Página detalhada de review |

## Começando

### Pré-requisitos

- Node.js 22+
- pnpm
- Conta no [Turso](https://turso.tech)

### Instalação

<Tabs>
  <TabItem label="Backend (ctech_be)">

    ```bash
    cd ctech/ctech_be
    pnpm install
    cp .env.example .env.local
    # Configure TURSO_DATABASE_URL e TURSO_AUTH_TOKEN
    ```

  </TabItem>
  <TabItem label="Frontend (ctech_fe)">

## Segurança

- CSP configurada via middleware (ctech_fe)
- Consultas parametrizadas ao banco (sem SQL injection)
- Cliente Turso usado apenas em SSR (nunca exposto ao cliente)
- HSTS, X-Frame-Options, X-Content-Type-Options
