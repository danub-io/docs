Welcome to the **CTECH (TechReveal)** documentation, an automated ecosystem for hardware analysis, comparison, and curation powered by AI.

This ecosystem is divided into **two projects** that share the same database:

<Card title="ctech_be (Next.js)">
Local tool that runs on your machine to populate the database. Product ingestion, prices, reviews, etc. No login needed, won't be deployed, only you use it.
</Card>

<Card title="ctech_fe (Astro + Cloudflare)">
Public site that reads from this database to display products, categories, and prices. It also writes user data (registration, login, Google OAuth, 2FA, user reviews).
</Card>

## Overview

CTECH is divided into two repositories that share the same **Turso (SQLite)** database:

| Repository | Stack | Role |
|-------------|-------|-------|
| **ctech_be** | Next.js 16, Turso, Pino, AI SDK, Tailwind v4 | Backend/Panel: Automation, Scrapers, AI Processing |
| **ctech_fe** | Astro 6, React 19, Tailwind v4, Turso, Zod v4 | Frontend/Public: High-performance interface optimized for SEO |

```
ctech_be (Server Actions) → Turso DB (SQLite) ← ctech_fe (Astro SSR)
```

## Architecture — Automation Modules

### Backend (ctech_be)

The backend is divided into 9 independent modules (M1-M9):

| Module | Name | Function |
|--------|------|---------|
| M1 | Entry | Ingestion and semantic duplicate detection via AI |
| M2 | Discovery | Technical review link search |
| M3 | Extraction | Review processing and spec extraction via AI |
| M4 | Consolidation | Review unification with Bayesian scoring |
| M5 | Prices | Price monitoring across stores |
| M6 | Auditing | Affiliate link and stock auditing |
| M7 | CMS | Central product catalog manager |
| M8 | Settings | Global settings panel (AI, Scrapers, Logs) |
| M9 | Documentation | Integrated Markdown documentation viewer |

### Frontend (ctech_fe)

| Route | Description |
|-------|-------------|
| `/` | Home with featured product and trends |
| `/category/[category]` | Recommendation guide hub by category |
| `/guide/[slug]` | Individual guide page with selected products |
| `/product/[slug]` | Detailed product page with specs, reviews, and prices |
| `/product/[slug]/reviews` | All press reviews for a product |
| `/product/[slug]/user-reviews` | All user reviews for a product |
| `/compare` | Side-by-side product comparison |
| `/community` | Community review feed |

## Getting Started

### Prerequisites

- Node.js 22+
- pnpm
- A [Turso](https://turso.tech) account

### Installation

<Tabs>
  <TabItem label="Backend (ctech_be)">

    ```bash
    cd ctech/ctech_be
    pnpm install
    cp .env.example .env.local
    # Configure TURSO_DATABASE_URL and TURSO_AUTH_TOKEN
    pnpm dev
    ```

  </TabItem>
  <TabItem label="Frontend (ctech_fe)">

    ```bash
    cd ctech/ctech_fe
    pnpm install
    cp .env.example .env
    # Configure TURSO_DATABASE_URL and TURSO_AUTH_TOKEN
    pnpm dev
    ```

  </TabItem>
</Tabs>

## Security

- CSP configured via middleware (ctech_fe)
- Parameterized database queries (no SQL injection)
- Turso client used only in SSR (never exposed to the client)
- HSTS, X-Frame-Options, X-Content-Type-Options
