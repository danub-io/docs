Welcome to the **Editora** documentation, a vibe coding tool for professional book production. Write chapters in Markdown and Editora handles the publishing workflow — AI review, typesetting, and export to PDF (print-ready) and EPUB.

---

## Overview

```
chapters/*.md → [AI: Edit/Review] → [Pandoc + LaTeX] → output/*.pdf / *.epub
```

### Technology Stack

| Tool | Role |
|------|------|
| **Python 3.12+** | Core language |
| **Typer** | CLI interface |
| **Pydantic** | Validation and settings |
| **Pandoc + LaTeX** | Professional typesetting |
| **LangChain** | AI orchestration |
| **LanguageTool** | pt-BR grammar checking |

## Features

| Feature | Description |
|---------|-------------|
| Markdown writing | Git-versionable chapters, easy to convert |
| AI editing | 3-level copyediting (light/medium/aggressive) |
| Consistency review | Detects contradictions, plot holes, timeline issues |
| Proofreading | pt-BR grammar and spelling correction |
| Typesetting | Print-ready PDF (KDP compliant) and EPUB |
| Intuitive CLI | Simple commands for the entire workflow |

## Getting Started

### Prerequisites

- Python 3.12+
- Pandoc
- LaTeX (TeX Live or TinyTeX)

### Installation

```bash
git clone <repo-url>
cd editora
python -m venv .venv
source .venv/bin/activate
pip install -e .
```

### Quick Start

```bash
editora init --title "My Book" --author "Your Name" --output my-book
cd my-book
# Write chapters in chapters/*.md
editora build    # Generates PDF + EPUB
```

## CLI Commands

| Command | Description |
|---------|-------------|
| `editora init` | Creates a new book project |
| `editora build` | Compiles to PDF/EPUB |
| `editora edit` | Edits chapters with AI |
| `editora proofread` | Reviews grammar and spelling |
| `editora consistency` | Checks global consistency |
| `editora template` | Generates LaTeX templates for formatting |
| `editora info` | Shows project statistics |
